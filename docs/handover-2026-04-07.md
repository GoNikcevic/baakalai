CREATE INDEX IF NOT EXISTS idx_memory_patterns_category ON memory_patterns(ab_category);
CREATE INDEX IF NOT EXISTS idx_memory_patterns_sectors ON memory_patterns USING GIN (sectors);
CREATE INDEX IF NOT EXISTS idx_memory_patterns_targets ON memory_patterns USING GIN (targets);

ALTER TABLE versions ADD COLUMN IF NOT EXISTS tested_steps TEXT[] DEFAULT ARRAY[]::TEXT[];

## 💡 Commentaires / décisions importantes

- **Demo mode** : on a eu un bug où la plateforme montrait des données démo même sans toggle. Fix appliqué à AppContext + MemoryExplorer + Recos + CRMAnalytics. Règle : aucune donnée démo ne doit jamais apparaître sans `localStorage.bakal_demo_mode === 'true'`.
- **Archivage** : préféré à la suppression pour garder l'historique intact (patterns memory, versions, diagnostics). Filtre par défaut `active` pour ne pas polluer la vue.
- **A/B : tested on E1 + LI1 only** : plus de tests simultanés = plus de variables non contrôlées = conclusions bruitées. Décision stratégique, pas technique. Extensible si besoin.
- **Split 50/50 natif Lemlist** : on délègue l'exécution à Lemlist et on se concentre sur la couche intelligence (gen + memory + analyse). Quand on ajoutera Apollo/Instantly, on délèguera aussi à leur split natif.
- **Mémoire collective** : anonymisée strictement (pas de verbatim, pas de client names, pas de sample exact). Décision : opt-in par défaut via CGU. Valeur = recommandations précises sur segment dès le 3e test concordant.
